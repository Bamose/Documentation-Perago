## FormBuilder Component

The FormBuilder component generates a form dynamically based on a configuration object passed to it.

## Functionality

**Input generation**

It generates inputs dynamically based on the `inputField.type` specified in each schema.

The supported types are:

- textinput
- textarea
- select
- checkbox
- radio
- passwordinput
- custom

For each type, there is a specific function that renders that input - like generateTextInput, generateSelectInput etc.

The generateForm method checks the type and calls the appropriate render method.

The render methods use React Hook Form to register each input and display errors.

**Multi-language support**

- It supports multi-language inputs by checking the `hasLocal` flag on the schema.
- If true, it renders inputs for each language in the `locale` array.
- The name is appended with the locale code like `fieldName.en`.

**Flexibility**

- The config driven approach allows generating any kind of form without touching the component code.
- Adding a new field is as easy as adding a new schema - no need to modify the component.
- The `custom` type allows rendering any custom component passed in config.

**Controlled inputs**

- React Hook Form provides full control over the inputs.
- Values and errors are managed by the parent component.
- Can integrates with form validation libraries easily.
- Helper methods like `append`, `remove` allows dynamic fields.
## Example

```js
const formConfig = {
  schemas: [
    {
      name: 'firstName',
      label: 'First Name',
      inputField: {
        type: 'textinput',
        control: {
          placeholder: 'Enter first name'
        }
      }
    },
    {
      name: 'age',
      label: 'Age',
      inputField: {
        type: 'numberinput',
        control: {
          placeholder: 'Enter age'
        }  
      }
    }
  ]
}

return (
  <FormBuilder
    config={formConfig}
    register={/* */} 
    errors={/* */}
    control={/* */}
  />
)
```
